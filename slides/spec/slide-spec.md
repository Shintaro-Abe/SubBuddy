## S4 iPhone→SubBuddy 利用データ（1:00）

- **見出し**：iPhoneの利用データを「集計値だけ」Macへ（2つの方法で実現）
- **図版（Napkin・横・light）**：2経路が SubBuddy に合流する流れ図。
  - 方法1: Screen Time(DeviceActivity)：対象アプリ利用→しきい値イベント→拡張→App Group→本体→HTTP POST
  - 方法2: Shortcuts：アプリを開く→オートメーション発火→HTTP POST
  - 両経路→「HTTP POST（集計値だけ＋トークン認証）」→ Mac: SubBuddy → Web画面に反映
  - 元 Mermaid（lines 102-114 of talk-draft）を Napkin で清書。
- **本文（各1〜2文）**：
  - 方法1：精密だが専用の仕組みが要る（利用時間がとれる）。
  - 方法2：iPhone標準だけで手軽だが粗い（最終利用日など）。
  - 共通：送るのは集計値だけ・事前共有トークンで認証＝プライバシー設計と一致。
- **ノート**：さらっと。使い分け（精密=Screen Time／手軽=Shortcuts）だけ。深掘りしない。

## S6 壁① 判定方式の考慮漏れ（1:40・山場）

- **見出し**：「どのサブスクを見直すべきか」の判定を、軽く考えすぎていた
- **本文（3ブロック固定）**：
  - **自分が甘かった点**：利用時間を測れば分かると単純化。妥当性／流し聞きの音楽・特典保有／登録直後のデータ無しを詰めていなかった。
  - **AIにどう助けられたか**：市場と技術を出典つきで検証→考慮漏れ続出。「利用時間で解約判定は前例ゼロ。iPhoneは対象アプリを暗号化して隠すため測りきれない」。別AIに反論もさせ「補助に下げるだけだと手動家計簿」と詰めの甘さも露呈。
  - **越えた結果**：「利用時間で測る」→「根拠の型に当てはめる方式」へ作り直し。中身は次スライド。
- **図版（Napkin・横・light）**：甘い見立て→AIで調べ直す/反論→考慮漏れを埋める→型に当てはめる方式へ（talk-draft lines 177-182 の流れ）。
- **ノート**：3ブロックの順で話すと一貫。

## S7 知見：6つの型と「利用」の定義（1:30）

- **見出し**：点数ではなく、状況の「型」に当てはめて理由を出す
- **補足**：判定は5種類だけ＝継続／様子見／ダウングレード検討／解約検討／強い解約候補。
- **図版（表1：6つの型＋未マッチは継続）**：

| 状況の型（＝そのまま理由になる） | アプリが出す判定 |
|---|---|
| 使っていない（能動利用なのに） | 様子見（長期間ずっと未使用なら強い解約候補） |
| 同カテゴリに複数あって割高 | 解約検討 |
| 同じサービスに安い有料プランがある | ダウングレード検討 |
| 同カテゴリに安い有料競合がある | 解約検討（理由：安い競合がある） |
| 年額更新が近い（7日以内） | 様子見（更新前に見直し） |
| 高額で長く継続（月¥2,000以上・12ヶ月以上） | 様子見（確認をうながす） |
| ——（どれにも当てはまらない） | 継続 |

- **図版（表2：「利用」の定義＝壁①の肝）**：
  - 能動利用（自分が使う）：前面で操作／裏で再生（音楽）／別端末で使う（PC・TV）
  - 受動利用：保管・同期・常時動くだけ（バックアップ等）
  - 権利保有：特典で持つだけ（プライム等）／容量で見るもの：iCloud+ 等
  - → 「使っていない」を出してよいのは**能動利用（前面・背景）だけ**。受動・権利保有には言わない。
- **ノート**：表は見せるだけ、読み上げない。押したら「利用の定義」に絞る。

## S8 壁② 実現可能性の見積もりの甘さ（1:30・山場）

- **見出し**：「利用量、取って送ればいいだけ」が甘かった
- **本文（3ブロック固定）**：
  - **自分が甘かった点**：取って送るだけと軽視。実際はApple特別権限／実機でしか確認不可／対象アプリは暗号化トークンで隠れる／Linuxコンテナではビルド不可／外部から手元Macへ安全に送る難しさ。
  - **AIにどう助けられたか**：実装前に実現方式を出典つきで調査・設計。やる/やめるの小さな技術検証として切り出し。送信は2経路（Screen Time／Shortcuts）＋集計値だけ＋トークン認証を設計。
  - **越えた結果（とAIに任せない線引き）**：iPhone→Macへ集計値が実機で届くまで動いた。最後の実機確認・OS許可操作は自分の手で。
- **図版**：S4 の2経路図を小さく再掲してもよい（成果の参照）。
- **ノート**：S4で先に成果、ここで「実はここに苦労」と振り返る流れ。
