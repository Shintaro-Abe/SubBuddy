# LT会話ログ匿名集計

このディレクトリには、会話本文を含まない匿名の集計結果だけを保存する。

## 現在の方法

- 対象期間：2026年6月27日〜2026年7月19日（草案）
- 単位：ユーザー発言
- 分類：ユーザー価値と技術方式の2軸を、ローカル言語モデルで2回判定
- 確定条件：温度0で分類定義・表示順を固定し、2軸を確認する順だけを逆にした2回の結果が完全一致
- 保存しないもの：発言本文、発言別ラベル、セッションファイル名、実在データ、認証情報

旧キーワード方式は未検出が72.2%あり、ユーザー目線の割合を推定する用途では不合格とした。旧方式の数値はLTで使用しない。

## 分類の実行

ローカルの`llama-server`を起動した状態で実行する。

```bash
node slides/lt-20260730/scripts/classify-sessions-local.mjs \
  --from 2026-06-27 \
  --to 2026-07-19
```

確定集計では`--to 2026-07-26`へ変更する。

## 合成分類テスト

```bash
node slides/lt-20260730/scripts/classify-sessions-local.mjs --synthetic-test
```

作り物の発言だけで、ユーザー価値、技術方式、両方、その他、短い承認の分類を確認する。

## 解釈上の注意

- `summary.json`はローカルモデルによる推定値であり、人手分類の正解データではない。
- 2回で不一致だった発言は主指標から除外し、未確定率を併記する。
- ユーザー価値と技術方式を両方含む発言は重複して数える。
- 「製品判断を含む発言」と「作業を進めるだけの発言」を分ける。
- 200文字を超える34発言（3.8%）は、先頭120文字と末尾80文字だけを分類へ使う。
