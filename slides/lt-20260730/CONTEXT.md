# SubBuddy LT

SubBuddyの開発過程を分析し、技術的な指示に自信がない人へ再現可能な作り方として伝えるための用語集。

## Language

**ユーザー目線の入力**:
困りごと、欲しい体験、優先順位、判断理由、利用時の違和感、生活上の制約について、ユーザーの立場からCodexへ伝えた内容。
_Avoid_: 非技術的な入力、素人の指示

**ユーザー価値**:
困りごと、欲しい結果・体験、機能要求、制約、優先理由、試作品への評価を含む入力。ローカル言語モデルで文脈を含めて推定する。
_Avoid_: 非技術的な入力、ユーザー目線シグナル

**技術方式**:
ユーザーが技術選定、アーキテクチャ、データ構造、API仕様、コード、実装方法、テスト方式を直接指定した入力。Codexが提案した技術と、技術名への単なる言及は含めない。
_Avoid_: 技術的な会話、実装に関する全入力、技術指定シグナル

**製品判断を含む発言**:
ユーザー価値または技術方式の少なくとも一方が確定した発言。作業を進めるだけの指示は含めない。
_Avoid_: 全ユーザー発言、全開発指示

**未確定**:
ユーザー価値と技術方式を確認する順番だけを変えた2回のローカル分類で、ラベルが一致しなかった発言。主指標の分母から外し、件数と割合を公開する。
_Avoid_: その他、無意味な入力

**Codexによる技術提案**:
ユーザー目線の入力を受けて、Codex側から提示した技術選定、設計または実装方法。ユーザーの技術的な入力とは区別する。
_Avoid_: ユーザーの技術指示

**Codexによる機能提案**:
ユーザーの課題・制約を受けて、Codex側から提示した必要な機能または体験の案。実装方法の提案とは区別する。
_Avoid_: Codexによる技術提案、ユーザー要求

**ユーザー判断**:
ユーザーが課題・制約を伝え、Codexの提案が自分にとって適切かを判断すること。技術的な正解を指定することとは区別する。
_Avoid_: 技術指示、Codexへの丸投げ

**ユーザー発言**:
ユーザーが一度に送信したメッセージ。現行集計の最小単位であり、文へ機械分割しない。
_Avoid_: 意味単位、文字数

**会話入力**:
ユーザーがチャットで直接Codexへ伝えた内容。リポジトリに保存済みの文書やコードは含めない。
_Avoid_: Codexへの全入力、プロジェクト文脈

**蓄積された技術的前提**:
Codexがリポジトリから参照できた技術スタック、設計、開発ルールなどの情報。会話入力とは別に集計・説明する。
_Avoid_: ユーザーの技術的な入力、Codexの事前知識

**対話型開発循環**:
ユーザー目線の入力をCodexが機能・設計・実装方針へ変換して文書に蓄積し、成果へのユーザー判断を次の入力にする循環。
_Avoid_: Codexへの丸投げ、一度きりのプロンプト
