# LTテーマコンペ結果

作成日: 2026-06-28

## 目的

SubBuddy を題材にした10分間LT会のテーマを、3チームで競争形式で検討した記録。

各チームは3名体制で検討した。

| 役割 | 責務 |
|---|---|
| スライド作成担当 | 10分LTのスライド構成・見せ方を整理 |
| 台本作成担当 | 話す順序・山場・言い切り方を整理 |
| テーマ/タイムキーパー担当 | テーマの焦点と10分内の収まりを管理 |

## 競争案

| チーム | テーマ | ファイル |
|---|---|---|
| チームA ローカル判断支援班 | 自動更新に流されないための、ローカルな見直し相棒を作る | `team-a.md` |
| チームB Steering Ops | 「AIに作らせる」から「AIを制御する」へ | `team-b.md` |
| チームC 信頼で見直す SubBuddy | サブスクを“解約させる”のではなく、後悔しない見直しを支えるローカルファーストUX | `team-c.md` |

## 投票結果

優勝: **チームA ローカル判断支援班**

理由:

- 既存の10分LTスライド資産と最も接続しやすい。
- SubBuddy のプロダクト価値、技術的な壁、AIとの開発過程を10分で過不足なく扱える。
- 聞き手にとって身近な「サブスク見直し」から入り、判定方式とiPhone連携の山場へ自然に進める。

詳細は `voting-result.md` を参照。
