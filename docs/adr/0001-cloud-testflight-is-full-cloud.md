# 小規模検証版はフルクラウド化する

SubBuddy の最初の配布版は、TestFlight で 20〜50 人程度に配る小規模検証版とする。この段階では、各ユーザーにローカル Mac サーバーを立ててもらう方式や VPN/Tailscale 前提のハイブリッド構成ではなく、Web・API・DB をクラウドに置き、iPhone アプリがクラウド API に集計値を直接送るフルクラウド構成を採用する。これは、商品版に近い認証・テナント分離・プライバシー説明・運用サポートの失敗パターンを早期に検証するためである。

**Considered Options**

- ローカル配布: 既存 MVP に近いが、DB・証明書・LAN・iPhone 送信設定がテスターごとに変わり、20〜50 人の検証にはサポート負債が大きい。
- ハイブリッド構成: 一部クラウド化しても、iPhone から届く受け口の API とユーザー別データ分離は必要になるため、検証したい商品版体験が曖昧になる。
- VPN/Tailscale 前提: 個人利用には成立しうるが、配布版の初期設定としては重く、既に不採用方針がある。

**Consequences**

クラウドに支出・利用状況を置くため、Apple サインイン、ユーザー別テナント分離、デバイス同期トークン、削除導線、プライバシーポリシー、保存時暗号化を小規模検証版の前提に含める。iPhone から送るデータは引き続き詳細ログではなく集計値のみに限定する。
