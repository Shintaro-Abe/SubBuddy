# 利用量同期の冪等はバケット最大値マージ＋当日レコード非削除

当日は利用時間が増えるにつれバケットが上がる（15分→30分→60分…）。送信の冪等キーは `(subscription_id, usage_date)` とし、バケットは新旧の**最大値**を採用してマージする（後勝ち上書きではない）。当日分は送信しても端末の App Group から削除せず、過去日（`usage_date < iPhone の今日`）かつ送信成功のものだけ削除する。`usage_date` は iPhone 現地時刻で確定し、サーバーは受け取った日付をそのまま保存する（サーバー時刻で日付を作らない）。

## Context / 背景

Spike の `UsageSyncService.syncAll()` は送信成功後に当日分も含めて削除しており、当日以降のしきい値到達（60分・120分）を送れず、当日の利用量を過小評価する既知欠陥がある。正式版ではこの挙動を本 ADR に沿って修正する。

## Considered Options

- 送信後に全削除（Spike のまま）：当日取りこぼしの既知バグを残すため棄却。
- 前日確定分のみ1日1回送信：重複は起きにくいが、当日の利用がダッシュボードに即反映されず体験が鈍いため棄却。

## Consequences

- サーバー側 upsert を「後勝ち」から「max マージ」に実装する必要がある。
- 当日を複数回送信しても最大値へ収束するため重複・過小計上を防げる。
