# 現行実装一覧

2026-07-23の`main`を、実在データや資格情報を読まずにコード・DBスキーマ・テスト監査台帳から確認した。

## Web・API・DB

- Web画面はサインインを含む11ページ。ホーム、契約、支出の内訳、見直し、更新間近、使い方、設定を提供する。
- APIは23経路・28操作。Apple認証、更新セッション、ログアウト、契約、支出、更新、見直し、利用量、案内進捗、端末、セッション、完全退会を扱う。
- Prismaは11モデル。利用者、案内進捗、端末、認証セッション、契約、請求、日別利用集計、見直しスナップショット、サービス、プラン、代替サービスを保持する。
- 1023px以下は上部バー・下部ナビ・「その他」を使い、1024px以上は左サイドバーを使う。同じHTMLを画面幅で再配置する。
- 320〜430px、横向き、200%相当、合成200契約、PC表示は自動試験済み。Safari・Chrome実機、VoiceOver、ソフトウェアキーボード、実機safe areaは未確認。

## iPhone

- 初回説明、Appleサインイン、最初の契約、年間支出、計測説明、最初の見直しを提供する。
- 主タブは「ホーム」「契約」「見直し」。設定から使い方、データの扱い、Screen Time、同期、端末・セッション、完全退会を確認できる。
- 契約CRUD、iCloud+容量、支出、更新間近、見直し、Screen Time対象選択、自動計測、自動同期を実装している。
- 通知設定には「更新日の事前通知」「同期失敗の通知」のトグルがあるが、画面全体が無効である。許可要求・予約・APNs配信は未接続で、通知は配信されない。
- データ出力と問い合わせ送信は利用可能な操作として表示しない。

## 見直し

- 計算結果には利用実績、単価、該当パターン、根拠、注意点、解約時・値下げ時の年間節約目安がある。
- スナップショットは利用実績、単価、`matchedPatterns`内の根拠・注意点を保存する。利用日・利用分・1利用日単価の固定値保存は修正済みである。
- 代替プラン・代替サービス・各節約額の専用保存項目はない。
- 4つの確認優先度への利用者向け表示移行、公式情報の90日鮮度、重大情報エラー時の表示停止は未完了である。

## 認証・Screen Time

- WebはHttpOnly CookieとCSRF、iPhoneはKeychainを使い、短期アクセストークンとローテーションする更新トークンを扱う。
- 端末・セッションの失効、他利用者データの拒否、配布版の固定ローカル利用者禁止を実装している。
- Screen Timeは1契約1アプリで、自動開始、対象変更・解除、認可復帰、契約削除時の後片付けを実装している。
- 起動、Appleサインイン完了、フォアグラウンド復帰時に集計値だけを自動同期する。通常同期だけでは見直しを自動再計算しない。
- 7日連続、実日付境界、配布候補のArchive・署名確認は残る。
