# 要求内容 — iPhone見直し案内とホームカードの状態修正

## 背景・目的

iPhoneの見直し画面で「この画面について」のボタンと説明が同時に見えることがある。また、使い方を終えた後もホームに通常の見直しカードが残るため、見直し詳細を確認しても未完了に見える。見直し画面には同じ目的の説明が2か所あり、ダークモードでは「この画面について」の再表示文字も見えづらい。

## 変更内容

- 「この画面について」は、説明または再表示ボタンの片方だけを安定して表示する。
- 見直し詳細を1件開けば、使い方の「見直し」は充足する既存条件を維持する。
- 見直し詳細を1件確認して使い方を終えた後は、ホームから見直しカードを外す。
- 契約ごとの閲覧済み状態や、自動的な順送りは追加しない。
- 見直し画面の常時説明を削除し、「この画面について」に説明を一本化する。
- 「この画面について」の再表示文字は、他画面と共通のダークモード対応色を使う。

## 受け入れ条件

- [x] AC-1: 見直し画面で説明表示中は再表示ボタンが出ず、閉じた後は再表示ボタンだけが出る。（実装・コード確認済み、実機未確認）
- [x] AC-2: 再表示ボタンを押すと説明が開き、閉じる操作を繰り返しても表示が競合しない。（実装・コード確認済み、実機未確認）
- [x] AC-3: 見直し詳細を1件開く既存操作で、使い方の見直し段階が完了する。（合成テスト追加・経路確認済み、Mac実行未確認）
- [x] AC-4: 見直し詳細を1件確認して使い方を終えた後は、ホームに「見直し内容を見る」カードを表示しない。（実装・残存文言確認済み、実機未確認）
- [x] AC-5: 見直し画面の「分かっている事実と分からない点…」という常時説明を削除し、同内容を重複表示しない。
- [x] AC-6: 見直し画面の「この画面について」はダークモードでも判読でき、他画面と同じ共通色・背景構造になる。（コントラスト確認済み、実機未確認）
- [x] AC-7: DB、API、契約・見直し判定、契約ごとの閲覧履歴を変更しない。

## 制約事項

- iPhoneの小さい画面とDynamic Typeで表示を固定高さにしない。
- 実在の契約名、見直し内容、利用状況をテストや証跡へ使用しない。
