# 利用送信ショートカットの「ひな形」の使い方

このフォルダの `subbuddy-usage.template.shortcut` は、**アプリを開いたら SubBuddy に「使ったよ」と知らせる**ための
ショートカット（＝iPhone の自動処理）の<ruby>雛形<rt>ひながた</rt></ruby>です。

アクション（＝処理の部品）を1個ずつ手で並べる代わりに、**部品を並べ終わった状態**を取り込めます。
あなたがやるのは、**空欄を3つ埋めるだけ**。秘密情報（IP やトークン）は入っていません。

> うまく取り込めないときは、無理せず手動で組む方法（`parallel-test-guide.html` の 5-1）に切り替えてください。
> このひな形は「時短のための補助」です。確実なのは手動です。

---

## このひな形がやってくれること

アプリを開くと、次の内容を Mac の SubBuddy へ送ります。

- 「このサブスクを、今日、開いた」という1件の知らせ
- 日付は **iPhone の今日の日付**を自動で入れる（毎回ちがう日付になる）

送る中身（イメージ。`<...>` はあとで埋める値）：

```json
{"items":[{"subscriptionId":"<サブスクID>","date":"<今日の日付>","used":true,"usageBucket":"none","source":"ios_shortcut"}]}
```

部品は3つ並んでいて、つなぎ込み（前の部品の結果を次へ渡す設定）は済んでいます。

1. **今日の日付を `2026-06-20` の形に整える**
2. **送る文章（上の JSON）を組み立てる**（日付は1の結果を差し込み済み）
3. **その文章を Mac の SubBuddy へ送る**

---

## 手順1：iPhone に取り込む準備をする

ひな形は Apple の「お墨付き（署名）」が付いていないため、**最初に許可設定が必要**です。

1. iPhone の **設定 → ショートカット** を開く。
2. **「信頼されていないショートカットを許可」を ON** にする。
   - このスイッチが見当たらないときは、**ショートカットアプリで何か1つ実行**してから再度開くと出ます。

## 手順2：ファイルを iPhone に送る

`subbuddy-usage.template.shortcut` を、次のどれかで iPhone に渡します。

- **AirDrop**（Mac → iPhone。一番かんたん）
- iCloud Drive に置いて iPhone の「ファイル」アプリで開く
- 自分宛てにメール添付して iPhone で開く

## 手順3：取り込む

iPhone で受け取ったファイルを**タップ** → 「ショートカット」アプリで開く → **取り込む（追加する）**。
取り込めない場合は、いったん中止して手動の方法に切り替えてください（冒頭の注意）。

---

## 手順4：空欄を3つ埋める（ここが本番）

取り込んだショートカットを開いて編集します。埋めるのは次の3か所だけ。
**`<...>` は山かっこごと消して、値だけ**を入れてください。

| 埋める場所 | 入れる値 | どこにある？ |
|---|---|---|
| `<MAC_IP>` | Mac の住所（例 `192.168.1.100`） | Mac で `ipconfig getifaddr en0` を実行すると出る |
| `<USAGE_SYNC_TOKEN>` | 送信のための合言葉 | Mac の `apps/web/.env` の `USAGE_SYNC_TOKEN=` の右側 |
| `<SUBSCRIPTION_UUID>` | どのサブスクか、を指す ID | SubBuddy 画面でサブスク詳細を開いた URL `…/subscriptions/xxx` の **`xxx`** |

**それぞれの場所：**

- `<MAC_IP>` … 3番目の部品「URL の内容を取得」の **URL 欄**の中。
- `<USAGE_SYNC_TOKEN>` … 同じ部品の **ヘッダ**（＝送信時の付箋）の `Authorization` の行。`Bearer ` の後ろに入れる。
- `<SUBSCRIPTION_UUID>` … 2番目の部品「テキスト」の文章の中。

> **注意：** サブスク ID は、**SubBuddy に今ある本物のサブスク**の ID にします。
> 存在しない ID だとエラー（送信失敗）になります。
> また、Screen Time の検証で使うサブスクとは **別のサブスク**にしてください（結果が混ざらないように）。

---

## 手順5：アプリを開いたら自動で動くようにする

「アプリを開いたら、このショートカットを動かす」という引き金（トリガ）は、
ショートカット本体には入れられない決まりです。**別に1つ作ります。**

1. ショートカットアプリ →「オートメーション」→ **個人用を新規作成**。
2. きっかけ：「**App**」→ 見守りたいアプリ →「**開いている**」。
3. 動かす処理：「**ショートカットを実行**」を選び、**さっき取り込んだひな形**を指定。
4. 静かに自動で動くように、次を設定：
   - 「実行前に尋ねる」**OFF**
   - 「実行時に通知」**OFF**
   - iOS 17 以降は「**すぐ実行**」を選ぶ

---

## 最後に：動くか確かめる

対象アプリを一度開いて、SubBuddy 側にその日の記録が入れば成功です。
詳しい確認の仕方（疎通の関門）は `parallel-test-guide.html` の 5-3 を見てください。

うまくいかないときの代表例：

| 様子 | 原因 | 直し方 |
|---|---|---|
| 「接続できない」になる | Mac の住所が違う／同じ Wi-Fi でない | `<MAC_IP>` を見直す。iPhone の Safari で `http://<住所>:3000` が開けるか先に確認 |
| 「権限がない（401）」 | 合言葉（トークン）が違う | `<USAGE_SYNC_TOKEN>` を `.env` と同じにする |
| 「サーバーエラー（500）」 | サブスク ID が SubBuddy に無い | 画面に出ている本物の ID を使う |

## 大事な約束

- このひな形は**秘密情報を含みません**。**値を入れたあとのファイルは、誰かに渡したり Git に保存したりしない**でください
  （住所・合言葉・ID は個人情報・秘密情報のため）。
- 取り込めても、**実機で開いて各部品の設定（特に3番目の「本文」が『ファイル』になっているか）を自分の目で確認**してから本番に使ってください。
