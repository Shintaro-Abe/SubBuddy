# 要求内容 — 利用量集計の自動同期

## 背景・目的

変更前のiPhoneアプリは、DeviceActivity Monitor Extensionが日別集計を端末内へ保存した後、利用者が設定画面で`今すぐ同期`を押さない限りクラウドへ送らなかった。一方、画面には「通常は自動で同期します」と表示しており、実装と説明が一致していなかった。計測開始・停止を自動化しても同期忘れで見直し結果が古いままでは、利用者が内部処理を管理する負担が残る。

本体アプリの起動、サインイン完了、フォアグラウンド復帰を契機に、端末内の日別集計を自動同期する。失敗した集計は削除せず、次の自動同期または手動再試行で送る。

## 変更・追加する機能の説明

### やること

- サインイン済みで本体アプリが利用可能になったとき、端末内の日別集計を自動同期する。
- アプリ起動時、Appleサインイン完了時、バックグラウンドからの復帰時を自動同期契機にする。
- 同時に複数の同期を実行せず、進行中の同期へ重ねて送らない。
- 通信失敗、認証切れ、サーバーエラーでは端末内集計を保持し、次回自動再試行できるようにする。
- 計測対象変更・解除の保留中は、その契約の集計を同期対象から外す。別契約の安全な集計は同期してよい。
- 同期成功日時、失敗状態、未送信件数を設定画面で確認できるようにする。
- `今すぐ同期`は通常操作ではなく、障害時・確認時の再試行手段として残す。
- 自動同期と手動同期は同じ`UsageSyncService`、デバイス同期トークン、日別upsertを使う。

### やらないこと

- DeviceActivity Monitor Extensionから直接通信しない。
- iOSが実行時刻を保証しない常駐・リアルタイム同期を約束しない。
- 今回はBGTaskSchedulerを追加しない。確実に説明できる起動・サインイン・フォアグラウンド復帰を自動契機とする。
- 日付境界、15分未満のしきい値、日をまたぐ連続利用の配分方式は変更しない。別作業で仕様確定・実機検証する。
- 詳細ログ、アプリ名、選択トークン、全アプリ一覧を送信しない。

## ユーザーストーリー

- 利用者として、SubBuddyを開くだけで新しい利用集計を同期したい。同期操作を覚えたくないため。
- 利用者として、通信できなかった集計を失わず、次に開いたとき自動で再送してほしい。欠損に気づかないまま見直したくないため。
- 利用者として、同期が遅れている場合だけ状態と再試行手段を確認したい。通常時に内部処理を操作したくないため。

## 受け入れ条件

- [ ] AC-1: サインイン済みでアプリが起動したとき、端末内に同期可能な集計があれば追加操作なしで同期を1回実行する。
- [ ] AC-2: Appleサインイン完了時とバックグラウンドからの復帰時に自動同期を試みる。
- [ ] AC-3: サインアウト中は同期せず、デバイス同期トークンがない状態で外部送信しない。
- [ ] AC-4: 同期中に別の自動同期契機が発生しても二重送信処理を開始しない。
- [ ] AC-5: 同期失敗時は端末内集計を保持し、次回の自動同期または`今すぐ同期`で再試行できる。
- [ ] AC-6: 対象変更・解除の保留中は対象契約を送らず、旧対象と新対象の集計を混在させない。
- [ ] AC-7: 成功時は最終同期確認日時と成功状態、失敗時は未送信保持と再試行可能状態を端末内へ記録する。
- [ ] AC-8: 設定画面で自動同期が有効であること、最終同期確認日時、未送信件数、失敗時の説明、手動再試行を確認できる。
- [ ] AC-9: 自動同期でも送信対象は日別・契約別の集計値だけで、詳細ログとアプリ選択情報を送らない。
- [ ] AC-10: 同じ日別集計を自動再送してもサーバー側で重複せず、短い利用段階へ戻らない。
- [ ] AC-11: 自動同期の成功・失敗・サインアウト・同時実行抑止・変更保留除外を合成テストと実機手順で検証できる。

## 制約事項

- 実在の契約、利用量、アプリ名、Appleアカウント、端末ID、認証情報をテスト・ログ・証跡へ残さない。
- iPhoneから送るのは日別集計値だけとし、DeviceActivity Monitor Extensionは通信しない。
- iOSのバックグラウンド実行は保証しないため、画面には「次回起動・復帰時に再試行する」と説明する。
- API、DBスキーマ、日別upsertキー`(subscriptionId, usageDate)`は変更しない。
