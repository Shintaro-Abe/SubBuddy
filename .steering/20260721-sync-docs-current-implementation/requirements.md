# 要求内容 — 現行実装と文書の同期

## 背景・目的

2026-07-20までに、iPhone主製品UI、Screen Timeの自動計測・自動同期、初回利用後の案内、Web設定・ログアウト、iPhone案内状態とダークモード操作色が実装された。一方、`docs/`、`manuals/`、`obsidian/`、`.steering/`の一部には、実装前の状態、完了済み確認を未確認とする記述、存在しない参照先が残っている。

コード、DBスキーマ、テスト監査台帳を根拠に、文書の役割を壊さず現行状態と一致させる。

## やること

- `docs/`の要求、機能設計、技術構成、構造、開発規約、用語を現行実装と照合する。
- `manuals/`のMarkdown正本を現行画面・操作・検証状態へ合わせ、変更した手順のHTMLを再生成する。
- `obsidian/`の技術メモへ現行性注記を反映し、完了済み事項と未確認事項を分離する。
- `.steering/README.md`を現行作業の索引へ更新し、関連する現行タスクリスト・移植表の実績を監査台帳と一致させる。
- 旧ステアリングの要求・設計は当時の判断記録として保持し、後続文書による上書きを索引または現行性注記で示す。
- ローカルリンク、コードフェンス、文書内の主要用語、生成HTML、機微情報形式を検査する。

## やらないこと

- コード、DB、API、画面、判定条件を変更すること。
- テスト監査台帳に証跡のない機能を「検証済み」「配布可能」へ引き上げること。
- 過去の承認内容を現行仕様に見せるため、旧ステアリングの要求・設計を無言で書き換えること。
- 実在の契約、利用量、アカウント、メールアドレス、識別子、資格情報を参照・記録すること。

## 受け入れ条件

- [x] AC-1: `docs/`の現行機能、API、データ、iPhone/Webの役割、実装状態がコード・スキーマ・監査台帳と一致する。
- [x] AC-2: `manuals/`の索引と手順が現行UI・運用方法を示し、存在しない参照先がない。
- [x] AC-3: 変更した手順書はMarkdownを正本としてHTMLを再生成し、内容とコピー対象数が一致する。
- [x] AC-4: `obsidian/`は調査時点の履歴を保ちつつ、現行適用範囲と後続実装による上書きを明示する。
- [x] AC-5: `.steering/README.md`が2026-07-21時点の正本関係を示し、過去文書を現行仕様として誤読しない。
- [x] AC-6: 関連タスクリスト・移植表の状態が`.audit/test-status.md`の実績を超えず、完了・未確認の表現が一致する。
- [x] AC-7: 対象Markdownの相対リンク切れ、閉じ忘れコードフェンス、廃止済み主要表現の不適切な現行記述が0件である。
- [x] AC-8: 変更対象に実在PII・資格情報・秘密鍵が含まれず、差分形式検査に合格する。

## 制約事項

- 現行仕様の正本は`docs/`、実装の正本はコードとDBスキーマ、試験結果の正本は`.audit/test-status.md`とする。
- 履歴文書の状態更新は、後続実装・試験の証拠を参照できる場合だけ行う。
- 文書だけの変更で実装上の保証範囲を広げない。
