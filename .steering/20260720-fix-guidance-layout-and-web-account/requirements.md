# 要求内容 — 案内レイアウトとWebアカウント操作の修正

## 背景・目的

iPhoneの「契約」で「この画面について」と表示フィルターが重なって見える。Webには既存のログアウトAPIがあるが、利用者が操作できる設定画面とボタンがない。本修正で、説明と操作の視認性を確保し、Webから安全にログアウトできるようにする。

## 変更内容

- iPhoneの契約画面で、説明と表示フィルターを間隔付きの縦並び領域へまとめる。
- Webサイドバーへ「設定」を追加する。
- Web設定画面にログアウト操作を追加し、成功後はサインイン画面へ移動する。
- ログインを使わないlocal modeでは、ログアウトボタンを表示せず説明を表示する。
- 既存のログアウトAPI、CSRF保護、セッション失効処理を再利用する。

## やらないこと

- 認証方式、Cookie、セッション期限、DB構造の変更。
- アカウント削除やプロフィール編集の追加。
- iPhoneの説明文や表示フィルターの機能変更。

## 受け入れ条件

- [x] AC-1: iPhoneの契約画面で、説明の表示・非表示にかかわらず表示フィルターと重ならず、間隔を保って縦に並ぶ。（実装・コード確認済み、実機未確認）
- [x] AC-2: Webの全ダッシュボード画面から「設定」へ移動できる。
- [x] AC-3: cloud modeの設定画面でログアウトすると既存APIが呼ばれ、セッションCookieが消去され、サインイン画面へ移動する。
- [x] AC-4: ログアウト失敗時は設定画面に再試行できるエラーを表示し、処理中の二重操作を防ぐ。
- [x] AC-5: local modeでは利用できないログアウト操作を表示せず、ログイン状態を使わないことを説明する。
- [x] AC-6: キーボード操作、狭いWeb画面、iPhoneの大きい文字でも主要操作へ到達できる構造を維持する。（iOS実機の大きい文字は未確認）

## 制約事項

- 実在のアカウント、メール、Cookie、トークンをテストや証跡に使用しない。
- ログアウトは既存の本人認証・Origin・CSRF検証を迂回しない。
- 前回の初回利用後ガイド実装と、作業対象外の未コミットファイルを保持する。
