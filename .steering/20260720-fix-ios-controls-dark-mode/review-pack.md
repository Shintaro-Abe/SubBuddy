## 前提・未決事項

### 要ユーザー判断

- なし。一般的なログインボタンを高さ44pt・最大幅320ptとして実装する。

### 設計上の前提

- 通常操作色の変更対象は、標準tintを使う通常ボタン、枠付きボタン、ツールバーである。
- 削除・警告・状態・グラフと、濃色カード上の明示的な白は対象外とする。

## 影響範囲

- `docs/`: 更新不要。
- iOSデザインシステム、サインイン画面、オンボーディング主要ボタン、退会前再認証。
- DB・API・認証処理・Web: 変更なし。

## セルフレビュー結果

| 観点 | 指摘 | 対応 |
|---|---|---|
| UX | Appleボタンを小さくしすぎると押しづらい | 高さ44ptを維持し、幅だけ320ptで抑える |
| Apple UI | 公式ボタンの最小寸法を下回る恐れ | 公式最小高さ30pt・幅140ptを上回る |
| アクセシビリティ | ダークモードで濃いブランド色が沈む | 明るい通常操作色へ切り替え、比率を計算する |
| UI | 明るいtintに白文字の塗りつぶしボタンは読めない | ダークモードだけ暗い文字へ切り替える共通スタイルを使う |
| セマンティクス | 全色を一括変更すると削除・警告の意味が崩れる | destructiveと意味色は既存指定を維持する |
| QA | 共通tint変更は影響範囲が広い | 置換対象、残存スタイル、ライト・ダークの比率を確認する |
