# 要求内容 — iPhone操作部品のサイズとダークモード修正

## 背景・目的

iPhoneの「Sign in with Apple」ボタンが大きく見える。また、ダークモードでは「この画面について」の「閉じる」をはじめ、通常操作の文字が背景へ沈む箇所が複数ある。共通デザインシステムでサイズと操作色を統一する。

## 変更内容

- Appleサインインボタンを高さ44pt、最大幅320ptの中央配置へ統一する。
- 通常操作の色をライト・ダークモードで切り替える共通色へ変更する。
- 塗りつぶしボタンは背景色に合わせて文字色も切り替え、コントラストを維持する。
- 通常のテキストボタン、枠付きボタン、ツールバー操作へ共通色を適用する。
- 削除操作の赤、見直しカード上の白、グラフ・状態表示色は変更しない。

## 受け入れ条件

- [x] AC-1: サインイン画面と退会前再認証のAppleボタンが、高さ44pt・最大幅320ptで中央表示される。
- [x] AC-2: ダークモードの「閉じる」「この画面について」など通常操作が背景に対して4.5:1以上のコントラストを持つ。
- [x] AC-3: 塗りつぶしボタンの文字がライト・ダークモードとも4.5:1以上で読める。
- [x] AC-4: 通常操作色を共通定義から適用し、画面ごとの色指定漏れを減らす。
- [x] AC-5: destructive、見直しカード、グラフ、状態色の意味を変更しない。
- [ ] AC-6: Dynamic Type、VoiceOver、44pt以上の主要操作領域を維持する。

> AC-6はコード上の44ptと既存ラベルを維持した。Dynamic TypeとVoiceOverの実動作は、Xcodeを使えるMac・実機で未確認。

## 制約事項

- Apple公式ボタンの外観要件と最小サイズを下回らない。
- 色だけで操作や状態を区別しない。
- 実在のアカウント、契約情報、認証情報をテストへ使用しない。
