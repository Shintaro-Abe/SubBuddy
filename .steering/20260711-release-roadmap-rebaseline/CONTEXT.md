# SubBuddyリリース検証

SubBuddyの小規模検証版で扱う中心価値と、検証時に混同してはいけない概念を定義する。

## Language

**中心価値**:
ユーザーがサブスク支出を把握し、優先して確認すべき契約とその根拠を理解して、自分で見直し判断を進められること。
_Avoid_: 解約判定、継続判定、自動意思決定

**見直し支援**:
契約の事実、確認の優先度、根拠、分からない点、選択肢を示し、ユーザー自身の判断を助ける情報提供。
_Avoid_: 解約レコメンド、最終判断、代行

**確認優先度**:
SubBuddyが持つ情報から、どの契約をいつ確認したいかを示す順序づけ。継続、変更、解約の結論ではない。
_Avoid_: 継続判定、解約判定、おすすめ結論

**補助手段**:
中心価値の実現を助ける機能。Screen Timeによる利用量計測、Web同期、レコメンド表示を含むが、それ自体を中心価値としない。
_Avoid_: 中心機能、検証目的

**有効参加者**:
募集条件を満たし、TestFlight参加に同意して、SubBuddyへの初回サインインを完了した人。契約登録や見直しへ進めなかった人も除外しない。
_Avoid_: アンケート回答者、見直し完了者、招待者全員

**一般公開移行基準**:
TestFlightの結果から、同じ製品方針で一般公開準備へ進めるかを判断するSubBuddy内部の基準。未達は開発中止ではなく、原因に応じた修正・再検証を意味する。
_Avoid_: Apple審査基準、TestFlight提出条件、開発完了条件
