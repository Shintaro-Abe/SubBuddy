# レコメンドは最終判断ではなく確認の優先度を示す

利用者向けの「継続・様子見・解約」という判定表示をやめ、「今確認したい」「更新前に確認したい」「情報が不足している」「現時点では急いで確認する材料が少ない」という確認優先度へ変更する。SubBuddyだけでは別端末・家族利用・将来の利用予定などを把握できず、継続・解約を断定すると誤解と不利益を生むためである。

## Consequences

- 詳細には事実、対象期間、根拠、分からないこと、誤判定の可能性、利用者が検討できる選択肢を表示する。
- 既存の内部判定コードは移行互換のため残せるが、利用者向け表示、成功指標、通知文では最終判断として使わない。
- 「現時点では急いで確認する材料が少ない」は継続推奨を意味しない。
