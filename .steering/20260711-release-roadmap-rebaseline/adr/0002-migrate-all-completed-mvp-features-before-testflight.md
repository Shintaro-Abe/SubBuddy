# 完成済みMVP機能を外部TestFlight前にiPhoneへ移植する

外部TestFlightの開始が遅くなることを受け入れ、現在のWebとiOS検証版で完成済みのMVP機能を、利用者向けiPhoneアプリへすべて移植してから外部テスターへ配布する。機能差による評価の混乱と移植漏れを避け、完成形に近い体験を検証するためである。請求履歴、自動抽出、AI補助、判断記録など未完成・将来候補の機能は対象に含めない。

## Consequences

- 実装と内部検証は、認証・契約・支出、見直し・更新、既存不具合修復、計測・単価、月次推移・容量管理、出力・削除・問い合わせ、全体回帰の順に分割する。
- 途中の内部ビルドは外部TestFlightの開始を意味しない。対応表の全項目と安全性ゲートを満たしてから外部配布する。
- 機能を減らして配布日を守る変更は、中心価値と既存機能維持への影響を示して再承認する。
