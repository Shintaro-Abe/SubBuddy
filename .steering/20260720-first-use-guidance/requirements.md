# 要求内容 — 初回利用後の使い方ガイド

## 背景・目的

iPhoneには、データの扱いの説明、Appleサインイン、最初の契約登録、支出表示、棚卸し、利用状況の説明、最初の見直しまでの初回導線がある。一方、初回導線を終えた後は「どの画面で何を確認し、次に何をすればよいか」が伝わりにくい。Webには専用の初回導線がなく、契約0件時の登録リンクだけである。

本作業では、iPhoneの既存初回導線を残したうえで、iPhoneとWebの画面サイズ・データ状態に合った案内を加える。説明を読むだけの連続ツアーではなく、実際の契約登録、支出確認、見直し確認、任意の利用状況設定へ直接進める案内とする。

## 変更・追加する機能の説明

### やること

- iPhoneの既存初回導線を維持し、途中で中断した場合は完了済みの操作を繰り返さず、未完了の手順から再開する。
- iPhoneのホームでは、新しい大きなチェックリストを追加せず、既存の「次に確認すること」カードに未完了の案内を1件だけ表示する。
- iPhoneの設定から、全手順と各機能の説明をまとめた「使い方」をいつでも開けるようにする。
- Webは、契約0件なら短い初回案内を表示し、既存契約があれば重複する初回導線を省略する。
- Webのホームには、画面幅を生かした「はじめ方」チェックリストを表示し、各項目から対象画面へ直接移動できるようにする。
- 契約登録、棚卸し完了、支出確認、見直し詳細の確認、利用状況の設定または見送りをもとに、進捗を自動判定する。
- 案内進捗をアカウント単位で保存し、iPhoneとWebで共有する。通信できないiPhoneでは端末内の進捗を使い、復旧後に安全に統合する。
- Screen Timeによる利用状況の計測は必須にしない。料金・更新日・重複などの見直しを先に体験し、対応する契約がある人だけ後から設定できるようにする。
- 「契約」「支出」「見直し」「更新間近」など、iPhoneとWebの利用者向け画面名を統一する。Webの「レコメンド」は「見直し」へ変更する。
- 各主要画面を初めて開いたときだけ、画面の目的と集計範囲を1〜2文で示す。説明は操作を遮らず、後から再表示できるようにする。
- 案内の改善に必要な完了状態だけを保存し、契約名、金額、利用時間などを案内分析用の別データとして収集しない。

### やらないこと

- iPhoneの既存初回導線を削除する、またはWeb用の画面へ置き換えること。
- 画面を順番に強制する吹き出しツアー、連続ダイアログ、動画チュートリアル。
- Screen Timeの許可や計測設定をSubBuddyの利用条件にすること。
- 契約、支出、見直しのドメインロジックや判定条件を変更すること。
- 外部の行動分析SDKを追加すること。
- 契約名、金額、更新日、利用量、Apple識別情報を案内の利用状況分析へ複製すること。

## ユーザーストーリー

- 初めて使う人として、初回登録後に次に確認する1項目を知りたい。小さいiPhone画面でも迷わず進めるため。
- iPhoneで使う人として、案内が通常の支出・見直し表示を圧迫しないでほしい。日常利用の邪魔にしたくないため。
- Webで使い始める人として、契約がない場合は登録から支出確認まで案内してほしい。空のダッシュボードで迷わないため。
- iPhoneですでに登録した人として、Webで同じ初回説明を繰り返したくない。すぐ現在のデータを確認するため。
- 途中で中断した人として、次回は未完了の手順から再開したい。同じ入力や説明を繰り返さないため。
- Screen Timeを使わない人として、計測を断っても支出と見直しを使い続けたい。機微な権限を利用条件にしたくないため。
- 使い方を忘れた人として、各画面の役割と全手順を後から確認したい。初回説明を覚えておく必要をなくすため。

## 受け入れ条件

- [x] AC-1: iPhoneの初回説明、Appleサインイン、契約登録、支出表示、棚卸し、最初の見直しを残し、利用状況の計測は見直し価値を確認した後の任意手順として案内できる。
- [x] AC-2: iPhoneで初回導線を中断・再起動した場合、保存済みデータと案内進捗から未完了の手順を求め、完了済みの入力・説明を強制的に繰り返さない。
- [x] AC-3: iPhoneのホームは既存の「次に確認すること」領域に「はじめ方 n/4」と次の1項目だけを表示し、新しい常設チェックリストで縦方向を圧迫しない。全項目は別の「使い方」画面で確認できる。
- [x] AC-4: Webは契約0件の場合だけ短い初回案内を表示し、1件以上ある場合は重複する初回導線を省略して現在の進捗を表示する。
- [x] AC-5: Webの「はじめ方」は、棚卸し、支出・更新日、見直し、任意の利用状況設定の状態と、対象画面へのリンクを一覧できる。完了後は折りたたみ、サイドバーの「使い方」から再表示できる。
- [x] AC-6: 契約の存在、棚卸し完了操作、支出画面の表示、見直し詳細の表示、利用状況の設定または見送りから進捗を自動判定し、利用者にチェック操作だけを要求しない。
- [x] AC-7: 同じアカウントの案内進捗はiPhoneとWebで共有され、同じ完了操作を両方で要求しない。オフライン中のiPhone進捗は失われず、通信復旧後の再送で重複や完了状態の巻き戻りが起きない。
- [x] AC-8: Screen Timeを許可・設定しない選択でも案内を完了でき、料金、更新日、重複、プラン情報による見直しを利用できる。Webは計測操作を提供せず、必要な場合だけiPhoneで設定できると示す。
- [x] AC-9: iPhoneとWebの利用者向け名称を「契約」「支出」「見直し」「更新間近」に統一し、内部のURL・型名・判定値は変更しない。
- [x] AC-10: 契約、支出、見直し、更新間近を初めて開いたとき、画面の目的と表示対象を短く説明する。説明は閉じられ、操作を遮らず、「この画面について」から再表示できる。
- [x] AC-11: 案内進捗APIは認証済み利用者本人の状態だけを読み書きし、入力値を固定したイベントとして検証し、再送しても同じ結果になる。案内進捗に契約名、金額、利用時間、メール、Apple識別情報を保存しない。
- [ ] AC-12: VoiceOver、最大Dynamic Type、iPhone標準画面・大画面、Webキーボード操作・200%拡大で案内内容と主要操作へ到達でき、合成データによる自動・画面・回帰確認が完了する。

## 制約事項

- iPhoneを主製品とし、WebやMacを利用条件にしない。
- iPhoneホームの主要情報量を増やさず、既存の「次に確認すること」領域を再利用する。
- 通常操作を遮るモーダル、連続ポップアップ、自動再生動画を使わない。
- 進捗はアカウント単位の最小データとし、詳細な行動履歴を作らない。
- iPhoneの端末内キャッシュには案内進捗だけを保存し、契約内容を追加複製しない。
- local modeでも同じ進捗APIとデータ構造を使い、固定ローカル利用者の進捗をMacローカルDBへ保存する。
- 実装・テスト・スクリーンショットには合成データだけを使う。
- iOS 17.4以降、iPhone専用・縦向き、日本語を初回対象とする。
