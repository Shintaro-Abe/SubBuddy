# 要求内容 — iOSアプリアイコン更新

## 背景・目的

SubBuddyの新しいファビコンと統一したブランドシンボルを、iPhone・iPadのアプリアイコンにも反映する。

## 変更内容

- やること: 確定済みの1024px PNGから、既存Asset Catalogが要求する18サイズを生成して置き換える。
- やること: `Contents.json`の対応関係と画像仕様を検証する。
- やること: 生成スクリプトの入力を新しい1024px PNGへ変更し、旧アイコンへ戻る再発を防ぐ。
- やらないこと: アプリの機能、画面、Bundle ID、表示名、Webファビコンを変更しない。

## ユーザーストーリー

- 利用者として、WebとiPhoneで統一されたSubBuddyのアイコンを見たい。

## 受け入れ条件

- [x] AC-1: AppIcon.appiconsetの18画像が新しいSubBuddyシンボルへ置き換わっている。
- [x] AC-2: 各画像のピクセル寸法が`Contents.json`のsize・scaleと一致する。
- [x] AC-3: 全画像がPNG・sRGB・不透明である。
- [x] AC-4: アプリのコード・設定・データに変更がない。
- [x] AC-5: 生成スクリプトが新しい正本PNGを入力として使い、旧「＋」アイコンを描画しない。

## 制約事項

- 実在のPII・機微データは扱わない。
- 元画像は1024×1024px、角丸なし、透明部分なしとする。
- iOSが表示時に角丸マスクを適用する前提を維持する。
