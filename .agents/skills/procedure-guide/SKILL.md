# 手順書（MD＋HTML）作成

初学者がそのまま操作できる手順書を作る。文章は AGENTS.md の応答スタイル（平易・簡潔・事務的）に従い、冗長な説明を省く。ミスしやすい点は簡潔に解説する。

成果物は2つ:
- **`<name>.md`** … 唯一の正（SSOT）。人が編集するのはこちら。
- **`<name>.html`** … MD から変換器で生成。単一ファイル完結・オフライン可・コピーボタン付き。

---

## 0. 不変条件（最優先）

- **MD が正（SSOT）。HTML は必ず `assets/md2guide.mjs` で MD から生成する。** HTML を手書きしない（ズレ防止）。内容修正は MD を直して再生成。
- **平易・簡潔**。専門用語は初出に「（＝平たい説明）」を添える。前置き・自己言及・繰り返しを書かない。
- **PII・シークレットを書かない**（AGENTS.md PII方針）。実際のトークン・APIキー・メールアドレス・本番値は手順書に載せない。プレースホルダ（`<発行したトークン>` 等）で示し、「コミットしない」注意を添える。
- **ミスしやすい点は `⚠`、確認は `✅`** で簡潔に。多用しない（本当に効く所だけ）。

---

## 1. 入力の取り込み

- **既存MDがある場合**：それを下敷きに、初学者向けに書き直す（後述の書式・変換規約に合わせる）。
- **自然言語の記述の場合**：手順をステップに分解して MD を起こす。
- 雛形は `assets/template.md` を複製して使うと早い。

不明点（保存先・対象OS・前提アカウント等）は、分かる範囲で要約してから不確かな点だけ確認する。

---

## 2. MD の書式（変換規約と一致させる）

変換器はこの規約で MD を解釈する。**ここを守らないと HTML のコピーボタン・GUI区別が出ない。**

### ステップの書式（統一）

各ステップは次の順で書く:

1. **○ 目的一言**（1行）— このステップで何をするか。
2. **□ 操作** — シェルコマンド or GUI番号手順。
3. **⚠ ミス注意**（必要なときだけ）。
4. **✅ 確認** — うまくいった状態。

### コピー単位＝シェル実行単位

- シェルコマンドは **```bash`（または `sh`/`shell`/`zsh`/`console`）フェンス**に書く。**1フェンス＝1コピーボタン＝1シェル実行（Enter 1回）**。
- 連続コマンドは、**途中で止まらず一気に実行してよいものだけ** `&&` で連結して1フェンスにしてよい。
- **ブラウザ起動・対話入力・出力確認を挟むコマンドは連結せず、別フェンスに分ける**（例：`node dist/index.js auth` のようにブラウザ認可が入るもの）。
- bash 以外の言語（```json` 等）のフェンスは表示のみ（コピーボタンは付かない）。

### GUI操作

- ブラウザ・アプリの操作は **番号付きリスト（`1.` `2.` …）** で書く。これが「GUI手順（コピー不可）」として描画される。

### callout

- ミス注意：`> ⚠ 本文`
- 確認：`> ✅ 本文`
- （任意の補足：`> 本文` … 通常 note）

### その他

- 見出し（`#`〜`####`）、箇条書き（`-`）、表（GFM `| … |`）、水平線（`---`）、インライン（`` `code` `` / `**bold**` / `[text](url)`）に対応。

## 3. HTML 生成（変換器の実行）

MD を書いたら変換器で HTML を生成する。**出力省略時は同じ場所・同名の `.html`。**

```bash
node /workspaces/SubBuddy/.agents/skills/procedure-guide/assets/md2guide.mjs <input.md>
```

出力先を変えるなら第2引数で指定:

```bash
node /workspaces/SubBuddy/.agents/skills/procedure-guide/assets/md2guide.mjs <input.md> <output.html>
```

- 変換器は依存ゼロ（Node 標準のみ）。Node 18以上。
- HTML は CSS/JS インラインの単一ファイル。ダブルクリックで開け、オフラインで動く。
- コピーは `navigator.clipboard`、非対応環境は `execCommand` フォールバック。成功時にボタンが「コピーした✓」へ一瞬変わる。

---

## 4. 品質チェック（生成後の自己点検）

- [ ] MD を直して**再生成**したか（HTML を手書きしていないか）。
- [ ] シェルの各フェンスが「1回で実行する単位」になっているか。止まる所で分かれているか。
- [ ] GUI操作が番号手順になっているか（コピーボタンが付いていないか）。
- [ ] ミス注意・確認が効く所に簡潔に置かれているか（多すぎないか）。
- [ ] PII・実シークレットが無いか。プレースホルダ＋「コミットしない」注意があるか。
- [ ] HTML が単一ファイルで開けるか（外部依存・CDN を使っていないか）。

---

## 5. 完了時

- 生成した MD と HTML のパスを提示する。
- HTML は「ブラウザでダブルクリックして開ける／コマンドはコピーボタンで1単位ずつコピーできる」旨を一言添える。
- コミットはしない（必要ならユーザーに確認）。
