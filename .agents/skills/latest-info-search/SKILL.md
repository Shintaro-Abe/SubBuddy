# Latest Info Search

## 目的

変わりやすい情報を、推測で答えずに検索・裏取りして返す。特に技術仕様、価格、法規制、ニュース、ライブラリ、API、製品情報、会社や人物の現状では一次情報を優先する。

## 基本方針

- まず「何が最新であるべきか」を特定する。日付・バージョン・地域・対象製品を曖昧にしない。
- 一次情報を優先する。公式ドキュメント、リリースノート、規制当局、企業公式発表、標準仕様、論文を先に見る。
- 重要な判断は複数ソースで照合する。食い違う場合は、どの情報が新しいか、どれが公式かを明記する。
- 技術質問では、ブログやまとめ記事より公式 docs / changelog / GitHub releases / RFC / issue を優先する。
- 実装判断では、外部情報だけで決めない。`package.json`、`docs/`、`.steering/`、`memory/` などローカル方針と照合する。
- PII・秘密情報・本番データを検索クエリや Gemini プロンプトに含めない。必要なら合成データ・一般化した説明に置き換える。

## ツール選択

### Codex Web検索

次のときは Web 検索を使う。

- 最新情報、価格、仕様、法規制、ニュース、リリース状況など、変わりやすい情報を確認する。
- 直接リンクや出典URLが必要。
- OpenAI 製品について調べる場合は、原則として公式 OpenAI ドメインに絞る。
- 技術仕様では、一次情報だけを根拠にする。

### Gemini Skill

次のときは `gemini` Skill を使う。

- 出典付きの軽い調査を別観点で行う。
- 既に作った見解の第二意見を取る。
- 長い記事や仕様の要約・翻訳をする。

Gemini が未認証なら止まり、`GEMINI_API_KEY` または OAuth 設定を案内する。認証を回避するためにローカルコードや秘密情報を渡さない。

### ローカル照合

実装やプロジェクト判断に関わるときは、検索結果の前後でローカル情報を確認する。

- 依存バージョン: `package.json`、lockfile
- プロジェクト方針: `AGENTS.md`、`docs/`、`.steering/`
- 既知の環境差分: `memory/`
- 既存実装: `src/`、テスト、設定ファイル

## 手順

1. 問いを絞る。
   - 対象、地域、日付、バージョン、用途を明確にする。
   - 曖昧でも合理的に進められる場合は、前提を明示して検索する。

2. 検索する。
   - 公式・一次情報を先に検索する。
   - 必要に応じてニュース、GitHub issue、Discussion、信頼できる二次情報で補う。
   - 出典の公開日・更新日を確認する。

3. 裏取りする。
   - 主要な主張ごとに出典を付ける。
   - 古い情報、新旧の食い違い、未確定情報を分ける。
   - 断定できない場合は「不確実」と書く。

4. ローカルと照合する。
   - 実装に関わる場合、現在のコード・依存バージョン・プロジェクト方針に合うか確認する。
   - 外部の最新手順が現在のプロジェクトにそのまま適用できない場合は、差分を説明する。

5. 結論を返す。
   - 結論を先に書く。
   - 要点は短く箇条書きにする。
   - 最後に出典URLを載せる。
   - 検索日を明記する。現在日はユーザー環境の日付を使う。

## 出力形式

基本形:

```markdown
結論: ...

要点:
- ...
- ...

注意点:
- ...

出典:
- https://...
- https://...
```

実装判断を含む場合:

```markdown
結論: ...

外部情報:
- ...

このリポジトリでの扱い:
- ...

次の作業:
- ...

出典:
- https://...
```

## 完了条件

- 最新性が必要な主張に出典URLが付いている。
- 公式・一次情報を優先したことが分かる。
- 不確実点や未確認点を隠していない。
- 実装判断ではローカル方針との整合を確認している。
