# 要求：Gemini 汎用相談スキル（軽量・高頻度）

## 背景
- 既存の `feature-research` スキルは「外部調査→Explore→Plan→codex反証→記録」の重い5ステップ定型フロー。実装戦略の立案向けで、起動コストが高い。
- 日常的には「ちょっと出典付きで調べたい」「第二意見が欲しい」「要約・翻訳して」程度の軽い用途が多く、もっと気軽に高頻度で叩ける窓口が欲しい。

## ゴール
- **Gemini CLI（無料 OAuth 枠）をブリッジする軽量スキル**を新設し、出典付きクイック調査を中心に、汎用の Gemini 相談窓口として使えるようにする。
- `feature-research`（重い・実装戦略立案）とは棲み分ける軽量版の位置づけ。

## 決定事項（ユーザー確認済み 2026-06-05）
- **形態**：精度と制御性を重視 → Skill（手動起動 `/gemini`）。
- **接続**：`@google/gemini-cli`（v0.45.2）＋ **無料 OAuth（Google ログイン）のみ**。
  - **API キー（`GEMINI_API_KEY` 等）は使用しない**（ユーザー方針 2026-06-05）。helper も API キー経路を持たない。
  - OAuth ログインは TTY＋ブラウザが必要なため、**VS Code の統合ターミナルで** `gemini` を対話起動して "Login with Google" を実施する（Claude の `! ` は非対話 TTY のため不可）。完了で `~/.gemini/oauth_creds.json` が生成され、以後ヘッドレス呼び出しが通る。
- **スコープ**：汎用 Gemini 相談窓口（出典付き調査が主。第二意見・要約・翻訳・ブレストも可）。
- **出力**：既定はチャット即返答。ユーザーが望めば `research/` 配下へ任意保存。

## 受け入れ条件
- `/gemini <問い>` で Gemini に問い合わせ、出典（URL）付きの簡潔な回答が返る。
- **認証は無料 OAuth のみ**。API キーは用いない。
- 認証未設定時は、VS Code 統合ターミナルでの OAuth ログイン手順を案内して安全に停止する（ハングしない）。
- Gemini にリポジトリのローカル実データ／コードを暗黙に読ませない（PII・機微データ方針の遵守）。
- 実装・コミット・ファイル編集は行わない（調査・相談のみ。読み取り専用で起動）。
- 速度優先のデフォルトモデル（flash）と、深掘り用モデル（pro）を切替可能。
