# Gemini 相談窓口（軽量・高頻度）

Gemini CLI（無料 OAuth 枠）をブリッジし、**出典付きのクイック調査**を中心に、第二意見・要約・翻訳・
ブレストなどを気軽に回すためのスキル。`feature-research` の重い5ステップに対し、こちらは**単発・軽量・高頻度**版。

---

## 0. 不変条件（最優先）

- **調査・相談のみ。コードの実装/変更/コミット/ファイル編集はしない。** gemini も `--approval-mode plan`（読み取り専用）で起動する。
- **Gemini にリポジトリのローカル実データ／コードを暗黙に読ませない。** helper は隔離した一時 cwd で起動し、workspace を渡さない（§4 PIIガード）。
- 検索クエリ・プロンプトに**実 PII・資格情報・社外秘を含めない**。例示は合成データで。
- `CLAUDE.md` のペルソナ運用・PII/機微データ方針・ローカルファースト・しきい値外出し方針を常に優先する。
- 重い実装戦略立案・既存コード探索が必要になったら、このスキルを終えて `feature-research` に切り替える。

---

## 1. 入力（問いの確定）

- **問いあり**（例：`/gemini Next.js App Router の Route Handlers でストリーミングする方法は？`）→ そのまま実行。
- **問いなし**で起動 → 「Gemini に何を聞きますか？（調査／第二意見／要約／翻訳 など）」と**1問だけ**確認。
- 広すぎる問いは、答えられる**単一スコープ**に軽く絞る（深掘りが必要なら feature-research を案内）。

---

## 2. モード選択（問いの性質で決める）

| モード | 用途 | プロンプトの組み立て方 |
|---|---|---|
| **research（既定）** | 出典付きで事実・最新情報を調べる | 「Web 検索で裏取りし、**主張ごとに出典 URL** を併記。不確実な点は『不確実』と明言。冗長禁止・結論先出し」を明記 |
| **second-opinion** | Claude の案への反証・別案 | Claude の案を**簡潔に要約して提示**し、「穴・前提誤り・別アプローチを指摘して」と依頼 |
| **summarize** | 与えた文章の要約 | 要約対象テキストを本文に貼り、出力長・観点を指定 |
| **translate** | 翻訳 | 原文と訳出方針（口調・専門用語の扱い）を指定 |
| **brainstorm** | 案出し | 制約・評価軸を与え、候補を箇条書きで |

ペルソナ：問いに応じて最適なものを選ぶ（例：Next.js 実装 → Next.js エンジニア、設計判断 → アーキテクト）。

---

## 3. 実行（helper 経由）

プロンプト本文を **stdin** から渡す。モデルは引数（`flash` 既定／深掘りは `pro`）。

```bash
# research（既定・高速）
cat <<'PROMPT' | .claude/skills/gemini/scripts/gemini-ask.sh flash
<ここに組み立てたプロンプト>
PROMPT

# 深掘り（推論を効かせたいとき）
cat <<'PROMPT' | .claude/skills/gemini/scripts/gemini-ask.sh pro
<ここに組み立てたプロンプト>
PROMPT
```

- helper は `--skip-trust --approval-mode plan -m <model> -o text` で gemini を**隔離 cwd**から起動する。
- **モデル指針**：高頻度・即答は `flash`。比較・設計判断・多段推論が要るときだけ `pro`。
- **精度を上げたいとき**：重要な事実は `flash` で一次回答 → 別クエリ（観点を変える／`pro`）で**裏取り**し、食い違いを明示する。

### research プロンプトの雛形（例）
```
あなたは該当分野の専門家です。次の問いに、Web検索で裏取りして回答してください。
- 結論を先に1〜3行で。続けて要点を箇条書き。各主張に [1][2]… の出典番号を付ける。
- **回答の最後に必ず「出典」セクションを設け、各番号に対応する実URLを一覧で列挙する（URLを省略しない）。** 一次情報（公式ドキュメント等）を優先。
- 不確実・情報が古い可能性がある点は「不確実」と明言。推測で断定しない。
- 冗長な前置きは不要。

問い：<ここに具体的な問い>
```

---

## 4. PII・機微データガード

- **隔離 cwd**：helper が一時ディレクトリで gemini を起動するため、リポジトリのファイルは Gemini の文脈に入らない。
- **コード文脈が要る場合のみ**：Claude 側で該当箇所を読み、**PII レビュー済みの抜粋**をプロンプト本文に直接貼る（実メール・実利用ログ・資格情報・本番データは貼らない／合成・一般化する）。
- 本番 DB・`.env`・`secrets/`・実領収書・実利用ログ・実メールアドレス等を gemini に渡さない。
- 機微情報に触れざるを得なくなったら、**作業を止めてユーザーに確認**する。

---

## 5. 出力

- **既定**：gemini の回答を Claude が確認し、**出典 URL 付きで簡潔に**チャット返答する（不確実点・裏取り結果も明記）。
- **任意保存**（ユーザーが望むときのみ）：
  ```
  research/[YYYYMMDD]-[topic-slug]/gemini-<topic>.md
  ```
  - `[YYYYMMDD]` は**ユーザー提示の現在日付**を使う（不明なら確認。推測しない）。
  - 冒頭に「調査日／使用モデル／問い／裏取り有無」を記す。`research/` が無ければ作成。

---

## 6. 縮退動作

- **認証未設定**（helper が exit 3）：ハングせず OAuth ログイン手順を案内して停止。
  - **VS Code 統合ターミナルで** `gemini` を対話起動 → "Login with Google"（無料枠 = Gemini Code Assist）。完了で `~/.gemini/oauth_creds.json` が生成され、以後ヘッドレス呼び出しが通る。
  - Claude の `! ` は非対話 TTY のため OAuth ログインは完了できない（手順案内のみ）。**API キーは使用しない方針**。
- **gemini 未インストール**：helper が `npx --yes @google/gemini-cli` にフォールバック（初回はダウンロードで遅い）。
- **レート上限/エラー**：無料枠超過時はその旨を伝え、時間をおく／`pro`↔`flash` 切替を提案。サイレントに省略しない。
- **無害なノイズ**：`[IDEClient] Failed to connect to IDE companion` / 末尾の `Invalid stream: ... empty response` / 稀に混ざる `</code>` は、exit=0 なら回答は完走しており無視してよい（必要なら helper 呼び出しを `2>/dev/null` で抑制）。

---

## 7. 完了時の振る舞い

- 「これは **Gemini への相談・調査**であり、コード実装はしていない」旨。
- research では**出典 URL** を必ず添える。裏取りの有無・不確実点を明記。
- 保存した場合は成果物パスを提示。
- 深い実装戦略が必要なら `feature-research` への切り替えを案内する。
