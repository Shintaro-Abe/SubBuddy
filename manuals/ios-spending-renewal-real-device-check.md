# iPhone支出・更新数値の実機確認手順

## 目的

iPhoneのホーム、支出の内訳、更新が近い画面が、Web版と同じAPIデータを正しく表示することを確認する。合成契約3件だけを使い、月払い・年払いの換算、カテゴリ別集計、月次推移、契約状態、14日以内の更新条件を確認する。

## 前提

- [iPhone利用者向けUIのRender実機確認手順](./ios-render-e2e-testflight-prep.md)が合格している。
- [iPhone契約CRUD・iCloud+容量の実機確認手順](./ios-contract-crud-real-device-check.md)が合格している。
- iPhoneとWeb版で同じテスト用Appleアカウントへサインインしている。
- 確認開始時点で、テスト用アカウントの契約が0件になっている。
- Renderへ確認対象の`main`がデプロイされている。

> ⚠ 実在の契約・金額・更新日を使わない。契約が残っている場合は、それが合成確認用だと確実に判断できるものだけを削除する。判断できない契約には触れず、この確認を中止する。

---

## 1. テスト監査台帳を実施中にする

○ 実機確認を始めたことを正本へ記録する。

□ 操作

1. `.audit/test-status.md`を開く。
2. 「iPhone支出・更新数値の実機API結合」の行を追加する。
3. 開始日、対象、確認内容を記入し、現在の結果を`実施中`にする。

> ⚠ 契約名、金額、Appleアカウント、識別子、スクリーンショットを台帳へ記録しない。

> ✅ 対象の行が`実施中`になっている。

---

## 2. 開始状態を確認する

○ 期待値を正確に計算できる状態にする。

□ 操作

1. Web版の契約一覧を開く。
2. 契約が0件であることを確認する。
3. iPhoneの`契約`タブを下へ引いて更新する。
4. iPhoneでも契約が0件であることを確認する。

> ⚠ 0件にできない場合、この手順の固定期待値は使えない。既存契約を無断で削除せず、確認を中止する。

> ✅ Web版とiPhoneの契約がどちらも0件になっている。

---

## 3. 月払いの合成契約を登録する

○ 月額換算と14日以内の更新対象を作る。

□ 操作

1. iPhoneの`契約`タブで`契約を追加`を押す。
2. 次の合成値を入力する。

| 項目 | 合成値 |
|---|---|
| サービス名 | `合成支出・月払い` |
| カタログ候補 | `手動入力` |
| カテゴリ | `video_streaming` |
| 日本円の請求額 | `1200` |
| 請求周期 | `月払い` |
| 更新日を登録 | オン |
| 次回更新日 | 実施日の7日後 |
| 契約状態 | `利用中` |
| 重要度 | `3 / 5` |

3. `保存`を1回だけ押す。
4. 契約一覧へ戻り、対象が表示されることを確認する。

> ✅ 月払い1,200円、利用中、7日後更新の合成契約を登録できる。

> ✅ 年払い12,000円、利用中、20日後更新の合成契約を登録できる。

## 5. 一時停止の合成契約を登録する

○ 支出と更新対象から除外される契約を作る。

□ 操作

1. `契約を追加`を押す。
2. 次の合成値を入力する。

| 項目 | 合成値 |
|---|---|
| サービス名 | `合成支出・一時停止` |
| カタログ候補 | `手動入力` |
| カテゴリ | `music` |
| 日本円の請求額 | `9000` |
| 請求周期 | `月払い` |
| 更新日を登録 | オン |
| 次回更新日 | 実施日の5日後 |
| 契約状態 | `一時停止` |
| 重要度 | `3 / 5` |

3. `保存`を1回だけ押す。
4. 契約一覧に合成契約が合計3件あることを確認する。

> ✅ 5日後更新でも集計対象外となる一時停止契約を登録できる。

---

## 6. ホームの合計を確認する

○ 利用中の月払い・年払いだけが年間支出へ反映されることを確認する。

□ 操作

1. iPhoneの`ホーム`タブを開く。
2. 画面を下へ引いて更新する。
3. `登録中の年間支出（目安）`が`26,400円`であることを確認する。
4. 件数が`2件`であることを確認する。
5. Web版のホームを再読込する。
6. Web版の月額合計が`2,200円`、年額合計が`26,400円`、継続中が2件であることを確認する。

計算根拠：

| 対象 | 月額換算 | 年額換算 |
|---|---:|---:|
| 月払い1,200円 | 1,200円 | 14,400円 |
| 年払い12,000円 | 1,000円 | 12,000円 |
| 一時停止9,000円 | 0円 | 0円 |
| 合計 | 2,200円 | 26,400円 |

> ✅ iPhoneとWeb版の年間支出・利用中件数が一致し、一時停止契約が除外される。

---

## 7. 支出の内訳を確認する

○ 月額換算とカテゴリ別集計を確認する。

□ 操作

1. iPhoneホームの`支出の内訳`を開く。
2. `月額換算`が`2,200円`であることを確認する。
3. `年額見込み`が`26,400円`であることを確認する。
4. カテゴリ別に、動画配信が`1,200円`、仕事効率化が`1,000円`と表示されることを確認する。
5. 音楽カテゴリの9,000円が表示されないことを確認する。
6. Web版の`支出`を開き、月額、年額、カテゴリ別金額がiPhoneと一致することを確認する。

> ✅ 月払いと年払いが正しく月額換算され、利用中だけがカテゴリ別に集計される。

---

## 8. 6か月推移を確認する

○ 登録月から積み上がる月次推移を確認する。

□ 操作

1. iPhoneの`支出の内訳`で`月次推移`を見る。
2. VoiceOverを一時的に有効にするか、棒グラフのアクセシビリティ情報を確認できる方法で各月の値を読む。
3. 直近6か月のうち、前月以前が`0円`、登録した当月が`2,200円`であることを確認する。
4. Web版の月次推移も、前月以前が`0円`、当月が`2,200円`であることを確認する。
5. iPhoneとWeb版で6か月の並びと値が一致することを確認する。

> ⚠ 月末・月初のUTC境界で確認する場合、当月ラベルが想定と異なる可能性がある。その場合はiPhoneとWeb版の一致を合格条件とし、勝手に期待値を書き換えない。

> ✅ iPhoneとWeb版の6か月推移が一致し、一時停止契約は推移にも含まれない。

---

## 9. 14日以内の更新を確認する

○ 契約状態と更新日による絞り込みを確認する。

□ 操作

1. iPhoneの`ホーム`へ戻る。
2. `更新が近い`の件数が`1件`であることを確認する。
3. `更新が近い`を開く。
4. `合成支出・月払い`だけが表示されることを確認する。
5. 20日後更新の`合成支出・年払い`が表示されないことを確認する。
6. 5日後更新でも一時停止中の`合成支出・一時停止`が表示されないことを確認する。
7. 表示された金額、請求周期、更新日が登録値と一致することを確認する。
8. Web版の`更新間近`を開き、対象が同じ1件であることを確認する。

> ✅ 利用中かつ14日以内の契約だけが、iPhoneとWeb版の両方に表示される。

---

## 10. 再読込後の安定性を確認する

○ 複数回読み込んでも件数や金額が重複しないことを確認する。

□ 操作

1. iPhoneホームを下へ引いて2回更新する。
2. 年間支出が`26,400円`、利用中が2件、更新が近いが1件のままであることを確認する。
3. `支出の内訳`を開き、月額換算が`2,200円`のままであることを確認する。
4. Web版も2回再読込し、同じ値であることを確認する。

> ✅ 再読込で契約や金額が重複せず、iPhoneとWeb版の数値が一致し続ける。

---

## 11. 合成契約を削除する

○ 確認用データを残さず終了する。

□ 操作

1. iPhoneの`契約`タブを開く。
2. `合成支出・月払い`を削除する。
3. `合成支出・年払い`を削除する。
4. `合成支出・一時停止`を削除する。
5. iPhoneの契約一覧が0件になったことを確認する。
6. Web版を再読込し、契約一覧が0件になったことを確認する。
7. iPhoneホームを更新し、年間支出0円、利用中0件、更新が近い0件になることを確認する。

> ⚠ 名前が完全に一致する3件だけを削除する。

> ✅ 合成確認用契約がiPhoneとWeb版の両方に残っていない。

---

## 12. テスト監査台帳を更新する

○ 合否と残った未確認範囲だけを記録する。

□ 操作

1. `.audit/test-status.md`の「iPhone支出・更新数値の実機API結合」の行を開く。
2. すべて成功した場合は`合格`へ変更する。
3. 一部だけ確認できた場合は`一部合格`とし、未確認の画面・条件だけを書く。
4. 失敗した場合は`不合格`または`もう一度確認が必要`とし、安全なエラー表示と操作名だけを書く。
5. 生ログ、契約内容、アカウント情報、識別子、スクリーンショットは記録しない。

> ✅ 結果、確認範囲、残った未確認範囲だけが台帳に記録されている。
