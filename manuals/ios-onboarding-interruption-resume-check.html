iPhone初回導線の中断・再開実機確認手順

目的

iPhoneの初回導線を途中で終了しても、未保存の入力を契約として登録せず、保存済みの合成契約から安全に再開できることを確認する。空状態、通信失敗、再試行、二重登録防止、初回完了状態の保持も確認する。

現行仕様では、未保存フォームの途中内容は端末へ保存しない。保存済み契約と案内進捗がある場合、アプリ再起動後は完了済みの手順を飛ばし、最初の全体像、最初の見直し、利用状況の説明のうち次に必要な画面へ自動で再開する。契約0件のときだけまず1件だけ登録しましょうを表示する。

前提

  • iPhone利用者向けUIのRender実機確認手順が合格している。
  • iPhone契約CRUD・iCloud+容量の実機確認手順が合格している。
  • iPhoneとWeb版で同じテスト用Appleアカウントへサインインできる。
  • テスト用アカウントの契約が0件である。
  • iPhoneでWi-Fiまたはモバイル通信を一時的にオフ・オンできる。
  • Renderへ確認対象のmainがデプロイされている。

実在の契約、金額、Appleアカウント情報を使わない。契約が残っている場合は、合成確認用だと確実に判断できるものだけを削除する。判断できない契約には触れない。

1. テスト監査台帳を実施中にする

○ 実機確認を始めたことを正本へ記録する。

□ 操作

  1. .audit/test-status.mdを開く。
  2. 「iPhone初回導線の中断・再開」の行を追加する。
  3. 開始日、対象、確認内容を記入し、現在の結果を実施中にする。

契約名、金額、Appleアカウント、識別子、スクリーンショットを台帳へ記録しない。

対象の行が実施中になっている。

2. 契約0件を確認する

○ 固定した条件で初回導線を確認できる状態にする。

□ 操作

  1. Web版の契約一覧を開く。
  2. 契約が0件であることを確認する。
  3. iPhoneの契約タブを下へ引いて更新する。
  4. iPhoneでも契約が0件であることを確認する。

0件にできない場合は、既存契約を無断で削除せず確認を中止する。

Web版とiPhoneの契約がどちらも0件になっている。

3. 初回導線を開始する

○ 認証を終了してからアプリを再インストールし、初回完了状態を安全にリセットする。

□ 操作

  1. iPhoneのホーム右上から設定を開く。
  2. このiPhoneからサインアウトを押す。
  3. Appleサインイン画面へ戻ることを確認する。
  4. iPhoneのホーム画面でSubBuddyを長押しする。
  5. アプリを削除を選び、確認画面でもアプリを削除を選ぶ。
  6. MacでXcodeのSubBuddyプロジェクトを開く。
  7. 接続した同じiPhoneを実行先に選ぶ。
  8. Xcode左上の実行ボタンを押し、SubBuddyを再インストールする。
  9. iPhoneで内容を確認して続けるを押す。
  10. 同じテスト用Appleアカウントでサインインする。
  11. まず1件だけ登録しましょうが表示されることを確認する。

サインアウトだけでは初回完了状態はリセットされない。必ずサインアウトしてからアプリを削除する。削除・再インストールはこの手順3だけで行い、手順6以降の中断確認では行わない。

再インストール後に説明画面、Appleサインイン、初回導線の順で進み、まず1件だけ登録しましょうが表示される。

4. 契約0件の案内を確認する

○ 0件を通信失敗と混同しないことを確認する。

□ 操作

  1. 通信が有効な状態で登録済みの契約を読み込むを押す。
  2. 読込中の表示が消えるまで待つ。
  3. このAppleアカウントに登録済みの契約は見つかりませんでしたという案内が表示されることを確認する。
  4. 契約登録画面へ自動で移動しないことを確認する。

契約0件が案内され、通信エラーや契約登録画面への誤遷移がない。

5. 通信失敗と再試行を確認する

○ 入力を失わずに通信を再試行できることを確認する。

□ 操作

  1. iPhoneのコントロールセンターまたは設定で、Wi-Fiとモバイル通信を一時的にオフにする。
  2. SubBuddyへ戻る。
  3. 登録済みの契約を読み込むを押す。
  4. 通信できない旨ともう一度試すが表示されることを確認する。
  5. Wi-Fiまたはモバイル通信を元に戻す。
  6. 通信が復旧したことをSafariなどで確認する。
  7. SubBuddyへ戻り、もう一度試すを押す。
  8. 契約0件の案内へ戻ることを確認する。

機内モードを使う場合は、確認後に必ず元へ戻す。エラー参照コードやネットワーク情報を証跡へ転記しない。

通信失敗を0件として扱わず、復旧後の再試行で正しい0件案内を表示する。

○ 保存前の入力が契約として登録されないことを確認する。

□ 操作

  1. 最初の契約を登録を押す。
  2. 次の合成値を入力する。

  1. 保存を押さない。
  2. iPhoneのAppスイッチャーを開く。
  3. SubBuddyを上へスワイプして完全に終了する。
  4. SubBuddyをもう一度起動する。

まず1件だけ登録しましょうへ戻り、未保存フォームの途中画面を表示しない。

7. 未保存契約が作られていないことを確認する

○ 中断時の入力がAPIへ送られていないことを確認する。

□ 操作

  1. 登録済みの契約を読み込むを押す。
  2. 契約0件の案内が表示されることを確認する。
  3. Web版の契約一覧を再読込する。
  4. Web版も0件で、合成初回・未保存が存在しないことを確認する。

iPhoneとWeb版のどちらにも未保存契約が作られていない。

8. 合成契約を保存して中断する

○ 保存後の契約を再起動後に読み込めることを確認する。

□ 操作

  1. 最初の契約を登録を押す。
  2. 次の合成値を入力する。

  1. 保存を1回だけ押す。
  2. 最初の全体像に進み、年間支出が14,400円であることを確認する。
  3. AppスイッチャーでSubBuddyを完全に終了する。
  4. SubBuddyをもう一度起動する。
  5. 保存済み契約が自動で読み込まれ、最初の全体像へ戻ることを確認する。

保存後に中断してもサインイン状態を保ち、保存済み進捗から安全に再開できる。

9. 保存済み契約から再開する

○ 保存済み契約を重複させずに初回導線を再開する。

□ 操作

  1. 最初の全体像が自動表示されることを確認する。
  2. 年間支出が14,400円のままであることを確認する。
  3. Web版の契約一覧を再読込する。
  4. 合成初回・保存済みが1件だけであることを確認する。
  5. 棚卸しを続けるを押す。
  6. 登録済みに対象が1件だけ表示されることを確認する。

保存済み契約1件から再開し、二重登録や金額の重複がない。

10. 棚卸し画面で再び中断する

○ 後半で中断しても保存済み契約を失わないことを確認する。

□ 操作

  1. 棚卸し画面を表示したまま、AppスイッチャーでSubBuddyを完全に終了する。
  2. SubBuddyをもう一度起動する。
  3. 保存済み契約と未完了進捗から最初の全体像が自動表示されることを確認する。
  4. 年間支出が14,400円であることを確認する。
  5. 今は1件で始めるを押す。

後半で中断しても保存済み契約を読み直し、初回導線を続けられる。

11. 初回導線を完了する

○ 見直し価値を確認してから、Screen Timeを使わずホームまで進めることを確認する。

□ 操作

  1. 最初の見直しが表示されることを確認する。
  2. 利用状況の説明へを押す。
  3. 利用状況を加えると判断材料が育ちますが表示されることを確認する。
  4. 今回は使わないを押す。
  5. ホーム、契約、見直しの3タブが表示されることを確認する。
  6. ホームの年間支出が14,400円、利用中が1件であることを確認する。

Screen Time未許可でも初回導線を完了し、主要3タブを利用できる。

12. 完了状態が再起動後も残ることを確認する

○ 初回導線を繰り返し表示しないことを確認する。

□ 操作

  1. AppスイッチャーでSubBuddyを完全に終了する。
  2. SubBuddyをもう一度起動する。
  3. 初回導線ではなくホームが表示されることを確認する。
  4. 契約タブに合成初回・保存済みが1件だけ表示されることを確認する。
  5. 初回導線で4段階を完了したため、ホームにはじめ方 n/4の案内カードが残っていないことを確認する。

初回完了状態が保持され、再起動後にホームへ戻る。

13. 完了後の使い方と画面説明を確認する

○ 初回導線を終えた後も、必要な説明へ戻れ、案内が通常画面を圧迫しないことを確認する。

□ 操作

  1. ホーム右上の設定を開く。
  2. 使い方を開き、棚卸し、支出・更新日、見直し、利用状況の4項目が確認済みになっていることを確認する。
  3. 見直しタブを開く。
  4. 説明が閉じている場合はこの画面についてを押し、説明だけが表示されて再表示ボタンと二重にならないことを確認する。
  5. 説明内の閉じるを押し、説明が消えてこの画面についてだけが表示されることを確認する。
  6. 契約と支出の内訳でも同じ開閉関係になることを確認する。

使い方を後から確認でき、説明と再表示ボタンが同時に表示されず、ホームに完了済み案内カードが残らない。

14. 合成契約を削除する

○ 確認用データを残さず終了する。

□ 操作

  1. iPhoneの契約タブを開く。
  2. 合成初回・保存済みの詳細を開く。
  3. この契約を削除を押す。
  4. 確認文を読み、契約を削除を押す。
  5. iPhoneの契約一覧が0件になったことを確認する。
  6. Web版を再読込し、契約一覧が0件になったことを確認する。
  7. iPhoneホームを更新し、年間支出0円、利用中0件になることを確認する。

名前が完全に一致する合成契約だけを削除する。

合成確認用契約がiPhoneとWeb版の両方に残っていない。

15. テスト監査台帳を更新する

○ 合否と残った未確認範囲だけを記録する。

□ 操作

  1. .audit/test-status.mdの「iPhone初回導線の中断・再開」の行を開く。
  2. すべて成功した場合は合格へ変更する。
  3. 一部だけ確認できた場合は一部合格とし、未確認の画面・条件だけを書く。
  4. 失敗した場合は不合格またはもう一度確認が必要とし、安全なエラー表示と操作名だけを書く。
  5. 生ログ、契約内容、アカウント情報、識別子、スクリーンショットは記録しない。

結果、確認範囲、残った未確認範囲だけが台帳に記録されている。
