# iPhoneでWeb版SubBuddyを確認する手順

> iPhoneのSafariとChromeで、画面のはみ出し、操作のしやすさ、文字拡大、VoiceOverを確認します。所要時間は1機種・2ブラウザで約30分です。

## 前提

- 確認対象のiPhone
- SafariとChrome
- 開発用または検証用のSubBuddyのURL
- 契約名や金額には合成データ（＝実在しないテスト用データ）だけを使う

> ⚠ 実際の契約、支出、メールアドレス、資格情報は入力・撮影・記録しないでください。

## 確認する画面

| 種類 | 画面 |
|---|---|
| 主要画面 | ホーム、契約、見直し |
| 補助画面 | 支出の内訳、更新間近、使い方、設定 |
| 入力・詳細 | 契約追加、契約詳細、契約編集、サインイン |

## 手順

### ステップ1：Safariで縦向きを確認する

主要画面と補助画面が、横スクロールなしで使えることを確認します。

1. iPhoneを縦向きにする。
2. Safariで検証用のSubBuddyを開く。
3. 画面下の「ホーム」「契約」「見直し」を順に押す。
4. 右上の「その他」を押す。
5. 「支出の内訳」「更新間近」「使い方」「設定」を順に開く。
6. 各画面を一番下までスクロールする。

> ✅ 確認：本文が左右で切れず、横スクロールが発生しない。下部ナビゲーションが本文を隠さない。

### ステップ2：契約の詳細・追加・編集を確認する

狭い画面でも入力と保存がしやすいことを確認します。

1. 「契約」を開く。
2. 既存の合成データを1件開く。
3. 詳細を一番下まで読み、「削除」が末尾の「その他の操作」にあることを確認する。
4. 「編集」を押す。
5. 文字入力時にiPhoneが自動で拡大しないことを確認する。
6. 「戻る」で契約一覧へ戻る。
7. 右上の「＋追加」を押す。
8. 合成データだけを入力し、保存ボタンまでスクロールする。

> ✅ 確認：追加・編集画面では下部ナビゲーションが表示されない。入力欄は1列で、保存ボタンを押しやすい。

### ステップ3：画面説明の表示を確認する

説明と「この画面について」ボタンが同時に重ならないことを確認します。

1. 「見直し」を開く。
2. 説明が表示されている場合は「閉じる」を押す。
3. 「この画面について」ボタンだけが表示されることを確認する。
4. そのボタンを押す。
5. 説明だけが表示されることを確認する。

> ✅ 確認：説明と説明を開くボタンは、常にどちらか一方だけが表示される。

### ステップ4：横向きを確認する

高さが短い状態でも移動とスクロールができることを確認します。

1. 「見直し」を開いたままiPhoneを横向きにする。
2. 一番下までスクロールする。
3. 「ホーム」「契約」「見直し」を順に押す。
4. 縦向きへ戻す。

> ✅ 確認：横向きでも横スクロールが発生せず、下部ナビゲーションが本文を隠さない。

### ステップ5：Safariで200%表示を確認する

文字を大きくしても、主要操作が使えることを確認します。

1. Safariのアドレスバーにあるページメニューを開く。
2. ページの表示倍率を200%にする。
3. ホーム、契約、見直し、その他メニューを順に開く。
4. 契約追加画面を開き、入力欄と保存ボタンを確認する。
5. 確認後、表示倍率を100%へ戻す。

> ✅ 確認：文字やボタンが重ならず、横スクロールなしで主要操作を完了できる。

### ステップ6：VoiceOverで主要操作を確認する

画面を見なくても、操作の名前と順序が分かることを確認します。

1. iPhoneの「設定」→「アクセシビリティ」→「VoiceOver」を開く。
2. VoiceOverをオンにする。
3. Safariへ戻る。
4. 右へスワイプし、「ホーム」「契約」「見直し」「その他」を順に読み上げることを確認する。
5. 「その他」を開き、各リンクと「閉じる」が読み上げられることを確認する。
6. 契約追加画面で、入力欄の名前と保存ボタンが読み上げられることを確認する。
7. 確認後、VoiceOverをオフにする。

> ⚠ VoiceOverがオンの間は、選択は1回タップ、実行は2回タップです。

> ✅ 確認：操作の名前が意味のある日本語で読み上げられ、読み上げ順が画面の並びと一致する。

### ステップ7：Chromeでも再確認する

ブラウザ差でレイアウトや操作が変わらないことを確認します。

1. Chromeで同じ検証用URLを開く。
2. ステップ1からステップ4を繰り返す。
3. Safariとの違いがあれば、画面名、向き、操作、発生結果を記録する。

> ✅ 確認：SafariとChromeの両方で、画面のはみ出しや操作不能がない。

## 複数のiPhoneで確認する場合

画面幅の異なる次の3群から、利用できる端末を最低1台ずつ選びます。

- 小さい画面：iPhone SE相当
- 標準画面：6.1インチのiPhone相当
- 大きい画面：PlusまたはPro Max相当

機種ごとの専用レイアウトはありません。同じレスポンシブUI（＝画面幅に応じて自然に組み替わる表示）を確認します。

## 困ったとき

- 表示が古い場合：SafariまたはChromeでページを再読み込みする。
- 下部ナビゲーションが見えない場合：契約追加・編集画面では仕様どおり非表示。閲覧画面で再確認する。
- 実データが表示された場合：スクリーンショットを撮らず、確認を止めて開発担当者へ連絡する。
